Agent Teams 활성화하기

Agent Teams는 실험적 기능이라 기본적으로 꺼져 있습니다. 한 줄만 추가하면 켜집니다.

1
settings.json에 환경변수 추가
▶

Claude Code 대화창에 아래를 그대로 붙여넣으세요. 자동으로 설정 파일을 수정합니다.

Claude Code에 붙여넣기
~/.claude/settings.json 파일에 env 항목을 추가해줘. CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS 값을 "1"로 설정해. 기존 설정은 유지하고 env만 추가하면 돼.
완료되면 settings.json이 이렇게 됩니다:
settings.json 결과
{ "permissions": { "defaultMode": "plan" }, "env": { "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1" } }
2
Claude Code 재시작
▶

설정을 적용하려면 Claude Code를 한 번 재시작해야 합니다.
VSCode 기준: Claude Code 패널을 닫았다 다시 열면 됩니다.

3
확인 — 작동 테스트
▶

아래 프롬프트를 입력해서 Agent Teams가 활성화되었는지 확인하세요.

테스트 프롬프트
간단한 테스트야. 리서치팀과 기획팀 2개 팀을 동시에 실행해줘. 리서치팀: 국내 AI 어시스턴트 시장 규모 조사 (3줄 요약만) 기획팀: AI 어시스턴트 핵심 기능 3가지 리스트업 각 팀이 병렬로 동시에 실행되는지 확인하고 싶어.
💡
여러 팀이 동시에 실행되면서 결과가 각각 돌아오면 성공입니다. 순차적으로 하나씩 실행되면 활성화가 안 된 것이니 1번부터 다시 확인하세요.

부르는 방식이 다릅니다. 상황에 맞게 쓰세요.

팀 직접 지정
AI 어시스턴트 도입하려고 해.

⚠️
팀 프로젝트는 팀 수만큼 비용이 곱해집니다. 단일 작업에는 쓰지 마세요.
한줄 판단: "이 작업에 팀 2개 이상의 결과물이 서로 필요한가?" → Yes면 팀 프로젝트, No면 1팀만 호출.
작동 방식
팀 프로젝트는 실제로 어떻게 돌아가나

Agent Teams를 켜면, 프로젝트를 던졌을 때 Claude가 중간 관리자 역할을 합니다.
단, 모든 것이 자동은 아닙니다. 아래 3단계를 거칩니다.

A
나: 프로젝트를 던진다
팀을 직접 지정해도 되고, 안 해도 됩니다
방법 1 · 팀 직접 지정
"AI 어시스턴트 도입해줘.
리서치팀, 기획팀, 법무팀
동시에 투입해."
방법 2 · Claude에게 맡기기
"AI 어시스턴트 도입하려는데,
어떤 팀이 필요할지 판단해서
팀 구성하고 진행해줘."
↓
B
Claude: 팀 구성을 제안한다
사용자 승인 전까지 실행하지 않습니다
CLAUDE 응답 예시
"이 프로젝트에 다음 팀을 투입하면 좋겠습니다:

  1. 리서치팀 — AI 솔루션 시장 조사 + 비용 비교
  2. 기획팀 — 기능 PRD + 사용자 시나리오
  3. 법무팀 — 개인정보보호법 + 이용약관 검토

리서치팀 결과를 기획팀과 법무팀이 참고하도록 하겠습니다.
이대로 진행할까요?"
↓
C
나: "응" → 이후는 자동
승인 후에는 팀 운영이 자동으로 돌아갑니다
작업 분배 자동
팀 간 소통 자동
의존성 관리 자동
완료 알림 자동
💡
핵심: 팀 투입 결정은 내가 한다. 투입 이후 운영은 Claude가 한다.
즉, "어떤 팀이 필요한지"는 판단해야 하지만 "팀끼리 어떻게 일할지"는 신경 쓸 필요 없습니다.
팀 조합 설계
내 업무에 맞는 팀을 직접 조합하세요

모든 프로젝트에 8개 팀을 전부 넣을 필요는 없습니다.
업무 성격에 따라 필요한 팀만 골라서 조합하는 것이 비용도 아끼고 결과도 좋습니다.

업무별 추천 팀 조합
업무 유형 추천 팀 조합 왜 이 조합인가
신규 기능 기획 리서치 + 기획 + 법무 시장 근거 → PRD 반영 → 규제 체크
투자 라운드 리서치 + IR + 법무 + 회계 시장 데이터 → IR 자료 → 계약 검토 → 재무 정리
파트너십 검토 리서치 + 법무 + 커뮤니케이션 상대방 조사 → 계약 리스크 → 회신 이메일
채용 프로젝트 기획 + 인사 + 커뮤니케이션 JD 설계 → 계약서/온보딩 → 채용 공고
분기 리뷰 회계 + IR + 리서치 손익 정리 → 투자자 업데이트 → 시장 변화 반영
신규 시장 진출 리서치 + 기획 + 법무 + IR 시장 분석 → 진출 전략 → 규제 확인 → 투자자 설득

프로젝트: AI 어시스턴트 도입

프롬프트 하나로 5개 팀이 동시에 움직이는 예시입니다.

프로젝트 프롬프트
우리 제품에 AI 어시스턴트 기능을 도입하려고 해. 아래 팀별로 동시에 작업 시작해줘. 리서치팀: 국내 AI 어시스턴트 솔루션 비교 (ChatGPT API vs Claude API vs 자체개발). 비용, 성능, 연동 난이도 기준. 기획팀: AI 어시스턴트 기능 PRD 작성. 임차인 문의 자동응답, 관리비 안내, 계약 관련 FAQ가 핵심 기능. 법무팀: AI 챗봇 도입 시 개인정보보호법 이슈 검토. 이용약관에 추가해야 할 조항 정리. IR팀: 투자자 업데이트에 넣을 "AI 기능 도입" 슬라이드 1장 초안. 커뮤니케이션팀: 기존 고객(건물주)에게 보낼 새 기능 안내 이메일 초안. 각 팀은 작업 중 다른 팀에 필요한 정보가 있으면 직접 공유해.

팀 간 직접 소통 — 이게 핵심 차이

2단계에서는 모든 정보가 대표(나)를 거쳐야 했습니다. Agent Teams에서는 팀끼리 직접 대화합니다.

리서치팀 → 기획팀 "Claude API가 가성비 1순위. 건당 비용 데이터 공유합니다. PRD에 반영해주세요."
법무팀 → 기획팀 "AI 응답에 '자동 생성된 답변입니다' 고지 문구 필수. PRD에 UI 요구사항으로 추가해주세요."
기획팀 → 커뮤니케이션팀 "핵심 기능 3개 확정했습니다. 고객 이메일에 이 기능 리스트 반영해주세요."
리서치팀 → IR팀 "AI 도입 시 운영비 30% 절감 데이터 찾았습니다. 투자자 슬라이드에 넣으세요."

이 소통은 자동으로 이루어집니다. 대표가 중간에서 전달할 필요 없이, 각 팀이 필요한 정보를 직접 주고받습니다.

비용 관리 팁
1. 팀원 수 제한 — 한 프로젝트에 3~5개 팀이 적정. 8개 전부 동시 투입은 비용 대비 효과가 떨어집니다.
2. 단순 작업은 2단계로 — "이메일 하나 써줘"는 서브에이전트면 충분합니다. Agent Teams는 여러 팀의 결과물이 서로 영향을 줄 때만 쓰세요.
3. Pro/Max 플랜 권장 — API 과금이 아닌 Claude Pro($20/월) 또는 Max($100/월) 요금제를 쓰면 토큰 걱정 없이 사용 가능합니다.
판단 기준
Agent Teams를 쓸 때 vs 안 쓸 때
Agent Teams가 효과적인 경우
  • 신규 기능 기획 (리서치+PRD+법무 동시)
  • 투자 라운드 준비 (IR+리서치+커뮤니케이션)
  • 신규 사업 검토 (시장조사+법무+재무 동시)
  • 분기 리뷰 (회계+IR+기획 한번에)
  • 팀 간 결과물이 서로 참조되어야 할 때
서브에이전트(2단계)로 충분한 경우
  • 이메일 한 통 작성
  • 계약서 하나 검토
  • 단일 주제 리서치
  • PRD 하나 작성
  • 팀 간 연결이 필요 없는 독립 작업
📚
정리: 프로젝트 하나에 2개 팀 이상의 결과물이 서로 영향을 주면 Agent Teams. 독립 작업이면 서브에이전트. 대부분의 일상 업무는 서브에이전트로 충분합니다.

시리즈 A 라운드 준비
리서치 + IR + 법무 + 회계
프롬프트
시리즈 A 라운드 준비를 시작하려고 해. 아래 팀별로 동시에 작업해줘. 리서치팀: 국내 프롭테크 최근 투자 동향 (2024-2026). 유사 스테이지 투자 사례 5건. IR팀: 투자자 1-pager 초안. TAM/SAM/SOM + 핵심 지표 + 투자 요청 내용. 법무팀: 투자계약서(SAFE/전환사채) 주요 조항 체크리스트. 창업자 불리 조항 리스트. 회계팀: 최근 6개월 손익 요약 + 번레이트 계산 + 런웨이 몇 개월인지. 각 팀은 다른 팀에 필요한 데이터 있으면 직접 공유해.
🏭
신규 B2B 파트너십 검토
리서치 + 법무 + 기획 + 커뮤니케이션
프롬프트
A사에서 API 연동 파트너십 제안이 왔어. 검토해줘. 리서치팀: A사 회사 정보, 재무 상태, 시장 평판 조사. 법무팀: 파트너십 계약 시 주의할 조항 (독점, 책임, 해지, 데이터 소유권). 기획팀: API 연동 시 우리 제품에 미치는 영향, 기능 변경 필요 사항 정리. 커뮤니케이션팀: A사에 보낼 긍정적 회신 이메일 + 미팅 요청 초안. 리서치팀 결과를 법무팀과 기획팀이 참고하도록 공유해.
