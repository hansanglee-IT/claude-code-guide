기본 개념
Claude는 뭔가

Claude는 Anthropic이라는 회사가 만든 AI입니다.
ChatGPT를 만든 OpenAI에서 나온 사람들이 세운 회사이고, "안전한 AI"를 핵심 가치로 내세웁니다.

구분 ChatGPT Claude
만든 회사 OpenAI Anthropic
핵심 모델 GPT-4o, GPT-4.1 Claude Opus, Sonnet, Haiku
강점 범용성, 플러그인 생태계 긴 문서 처리, 정확한 지시 이행, 코드
웹 서비스 chat.openai.com claude.ai
LLM이 뭔데?

Claude든 ChatGPT든, 핵심 기술은 LLM(Large Language Model, 대규모 언어 모델)입니다.
쉽게 말하면 "엄청나게 많은 텍스트를 학습해서, 다음에 올 말을 예측하는 AI"입니다.

생태계
다 Claude인데, 뭐가 다른 건가

"Claude"라는 이름이 붙은 제품이 여러 개입니다. 각각 용도가 다릅니다.

비개발자 추천
claude.ai
브라우저에서 바로 쓰는 채팅형 AI. ChatGPT와 같은 포지션.
설치 없이 바로 시작할 수 있어 비개발자에게 가장 적합합니다.
결과물도 만들어주고(Artifacts), 파일 업로드도 가능.

한 마디로: 브라우저 안에서 끝나는 AI
이 가이드의 주인공
Claude Code
내 컴퓨터 안에 들어와서 일하는 AI.
파일을 직접 읽고 참조 → 맥락을 파악해서 더 정확한 결과물.
긴 문서도 잘리지 않고 파일로 바로 저장.
반복 업무를 한 줄 명령으로 자동화 (Skills, 에이전트).

단, VSCode 설치 + 초기 세팅이 필요합니다 (이 가이드가 도와줍니다)

할 수 있는 것: 이동 중 간단한 질문, 대화
🔔
핵심 차이: claude.ai는 브라우저 안에서 끝나는 AI. Claude Code는 내 컴퓨터 안에 들어와서 일하는 AI. 같은 Claude인데 할 수 있는 범위가 다릅니다.
claude.ai vs Claude Code — 구체적으로 뭐가 다른가
기능 claude.ai (웹) Claude Code
대화/질문 O O
파일 읽기 업로드한 파일만 내 컴퓨터 전체
파일 생성/수정 X O (자동 저장)
웹 검색 제한적 O
반복 업무 자동화 X O (Skills, 에이전트)
여러 팀 동시 실행 X O (Agent Teams)
설치 필요 없음 (브라우저) VSCode + 확장 프로그램
비교
Claude Code vs 다른 AI 도구들

"AI 코딩 도구"라고 불리는 것들이 많습니다. 각각 타겟이 다릅니다.

도구 주 타겟 핵심 용도 비개발자 적합도
ChatGPT 일반 사용자 대화, 글쓰기, 질문 높음
claude.ai 일반 사용자 대화, 글쓰기, 문서 분석 가장 높음
Cursor 개발자 코드 작성 특화 에디터 낮음
Windsurf 개발자 코드 작성 + AI 에이전트 낮음
GitHub Copilot 개발자 코드 자동완성 낮음
Claude Code 개발자 (비개발자도 세팅하면 가능) 코드 + 문서 + 리서치 + 자동화 낮음 (세팅하면 가능)
그러면 비개발자는 뭘 써야 하나?
⚠️
솔직한 답: 단순 질문/글쓰기만 할 거면 claude.ai가 더 편합니다.
Claude Code는 초기 세팅(VSCode 설치, 터미널 사용)이 필요하기 때문에 진입장벽이 있습니다.
그럼에도 내가 Claude Code를 사용하는 이유
1. 결과물이 파일로 바로 저장됩니다 — claude.ai는 복사-붙여넣기가 필요하지만, Claude Code는 파일로 바로 저장. 30페이지 분량도 잘리지 않습니다.
2. 내 파일을 직접 읽고 참조합니다 — 계약서 10개를 폴더에 넣고 "리스크 조항 뽑아줘"하면 다 읽고 정리. claude.ai는 하나씩 업로드해야 합니다.
3. 반복 업무를 한 줄로 실행합니다 — "/research 경쟁사 분석"처럼 단축 명령어를 만들 수 있습니다. claude.ai에서는 매번 처음부터 설명해야 합니다.
4. AI 팀을 구성할 수 있습니다 — 리서치팀, 법무팀, 기획팀을 역할별로 만들어 동시에 일을 시킬 수 있습니다.
5. 세팅은 한 번이면 됩니다 — 이 가이드를 따라 세팅하면, 이후에는 claude.ai만큼 편하게 쓸 수 있습니다.
용어 사전
시작 전에 알아야 할 단어들

1단계 가이드에서 이 단어들이 나옵니다. 외울 필요 없고, 나중에 헷갈리면 여기로 돌아오세요.

작업 환경
VSCode
Visual Studio Code
Microsoft가 만든 무료 코드 편집기. Claude Code는 여기에 설치합니다. 개발자뿐 아니라 문서 작업에도 쓸 수 있습니다.
터미널
Terminal
텍스트로 컴퓨터에 명령을 내리는 창. VSCode 안에 내장되어 있어서 따로 열 필요 없습니다. Claude Code가 여기서 실행됩니다.
IDE
Integrated Development Environment
통합 개발 환경. VSCode가 대표적인 IDE입니다. "IDE에서 실행하세요" = "VSCode에서 하세요"와 같은 말입니다.
확장 프로그램
Extension
VSCode에 기능을 추가하는 플러그인. Claude Code도 VSCode 확장 프로그램으로 설치합니다. 앱스토어에서 앱 설치하는 것과 같습니다.
Claude Code 핵심 개념
CLAUDE.md
Claude Memory Document
Claude에게 "나는 누구인지, 어떻게 일하고 싶은지"를 알려주는 파일. 한 번 작성하면 매번 반복 설명이 필요 없습니다.
Skills (스킬)
/research, /prd, /email 등
자주 하는 업무를 단축 명령어로 만든 것. "/research 경쟁사 분석"처럼 한 줄로 실행합니다.
서브 에이전트
Sub Agent
역할과 전문성을 가진 AI 팀원. "리서치팀", "법무팀"처럼 각자 맡은 일이 정해져 있습니다.
Agent Teams
에이전트 팀
여러 서브 에이전트가 동시에 일하며 서로 소통하는 기능. 프로젝트 단위로 팀을 투입할 때 씁니다.
자주 나오는 기타 용어
프롬프트
Prompt
AI에게 하는 질문이나 지시. "시장 리서치 해줘"가 프롬프트입니다. 잘 쓰면 결과가 좋아집니다.
토큰
Token
AI가 텍스트를 처리하는 단위. 한국어 1글자 = 약 2~3토큰. 토큰을 많이 쓸수록 비용이 올라갑니다.
컨텍스트
Context
AI가 한 번에 기억할 수 있는 대화량. Claude는 약 20만 토큰(책 1권 분량)까지 기억합니다.
MCP
Model Context Protocol
Claude Code를 Gmail, Notion, Slack 같은 외부 도구에 연결하는 기술. 4단계에서 다룹니다.

사용량 리셋은 어떻게 되나?
자정 리셋이 아닙니다. 한도에 도달한 시점부터 약 5시간 후 회복됩니다. 한도 초과 시 차단이 아니라 일시 대기이며, 추가 과금은 없습니다. Opus(고성능 모델) 사용 시 한도가 더 빨리 소진됩니다.

사용량 확인: 남은 사용량을 미리 확인하는 기능은 아직 없습니다. 한도에 도달하면 리셋 시각이 표시되니, 그때까지 대기하면 됩니다.
