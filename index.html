핵심 차이
왜 웹 Claude가 아닌 Claude Code인가

웹 Claude도 강력합니다. 그런데 비개발자 대표가 실무에서 쓰다 보면 반드시 벽에 부딪힙니다.
Claude Code는 그 벽을 없앤 버전입니다.

🧠
기억 — 매번 설명 안 해도 됩니다
웹 Claude는 창을 닫으면 초기화됩니다. "나는 스타트업 대표야, 한국어로 핵심만 말해줘"를 매번 반복해야 합니다.
Claude Code는 CLAUDE.md에 나를 써두면 영구적으로 기억합니다. 한 번만 세팅하면 됩니다.
💾
저장 — 결과가 실제 파일로 남는다
웹 Claude에서 받은 리서치 결과, 기획서 초안 — 전부 대화창 안에 있습니다. 창 닫으면 찾기 어렵습니다.
Claude Code는 결과를 내 컴퓨터 파일로 자동 저장합니다. 언제든 열 수 있습니다.
⚡
명령어 — 반복 작업을 단어 하나로
매번 긴 프롬프트를 작성하거나, 이전 프롬프트를 찾아서 붙여넣는다. 시간도 낭비고 퀄리티도 들쭉날쭉하다.
/research, /investor, /prd — 단어 하나로 전문적인 형식의 문서가 나온다.
🔗
한계 없음 — 긴 작업도 끊기지 않습니다
웹 Claude는 대화가 길어지면 앞 내용을 잊기 시작한다. 복잡한 작업은 중간에 리셋해야 한다.
Claude Code는 로컬에서 실행되어 훨씬 긴 작업도 맥락을 유지한다.
"웹 Claude는 호텔 방이다 — 깔끔하고 세팅돼 있지만, 올 때마다 처음부터 시작한다. Claude Code는 AI가 있는 내 아파트다. 커스터마이즈하고, 쌓아가고, 함께 만들어나갈 수 있다."
— Dan Shipper, Every CEO (Claude Code 비개발자 활용 리서치)

핵심 개념
Claude Code가 돌아가는 방식 — 4가지만 알면 됩니다

설치하고 쓰기 전에, 이 4가지 개념만 잡아두시면 나중에 막힐 때 스스로 해결할 수 있습니다.
개발 지식 없이도 이해할 수 있는 수준으로 설명합니다.

🧠
항상 적용
CLAUDE.md — Claude의 장기 기억

Claude Code를 열 때마다 자동으로 읽는 파일입니다. 여기에 "나는 스타트업 대표이고, 항상 한국어로 핵심부터 말해줘"라고 써두면 매번 설명할 필요가 없습니다.

무엇을 넣어야 하나: 나는 누구인가, 항상 지켜야 할 응답 원칙, 자주 하는 작업 목록. 단, 너무 많이 넣으면 오히려 효과가 줄어듭니다. 300단어 이내를 권장합니다.

비유하자면 매일 아침 팀원에게 "오늘도 잘 부탁해, 나는 이런 사람이고 이렇게 일하는 걸 좋아해"라고 브리핑하는 것 vs. 그 팀원이 이미 나를 완전히 알고 있는 것의 차이입니다.
⚡
필요할 때 호출
Skills — 반복 작업의 명령어화

/research, /investor, /prd 같은 명령어를 직접 만드는 것입니다. 각 명령어에는 "이 작업을 할 때 어떤 형식으로 어떻게 해줘"라는 지시문(SKILL.md)이 담겨 있습니다.

CLAUDE.md와의 차이: CLAUDE.md는 항상 켜져 있는 규칙이고, Skills는 필요할 때만 불러오는 전문 도구입니다. 리서치할 때만 /research를 쓰는 것처럼요. 공식 문서 기준으로 Skills는 ~/.claude/skills/ 폴더 안에 폴더명/SKILL.md 구조로 저장됩니다.

실전 팁 같은 작업을 세 번 이상 반복하게 된다면 그게 Skills로 만들 신호입니다. 매번 프롬프트를 새로 쓰는 건 시간 낭비입니다.
🔄
작동 방식

Claude Code는 내 컴퓨터에서 직접 실행됩니다. 파일을 읽고, 쓰고, 저장하고, 인터넷을 탐색합니다. 웹 Claude가 대화창 안에서만 존재하는 것과 다릅니다.

고급 활용 — 별도 가이드
서브 에이전트 — 분리된 작업자

서브 에이전트는 Claude Code 안에서 특정 역할을 맡아 독립적으로 일하는 Claude입니다. 메인 대화창을 복잡하게 만들지 않고, 무거운 작업을 따로 처리한 뒤 결과만 돌려줍니다.

스타트업 대표 관점으로 이해하기:
혼자서 리서치도 하고, 계약서도 보고, 기획서도 쓰는 게 지금 방식입니다. 서브 에이전트는 각 역할을 전담하는 Claude를 만드는 것입니다. "국내 SaaS 시장 분석해줘"라고 입력하면, Claude가 description을 보고 리서치 전용 에이전트에 자동으로 위임해서 독립적으로 조사하고 결과만 돌려줍니다. 대표는 지시만 하면 됩니다.

비유하자면 Skills가 "업무 매뉴얼이 있는 도구"라면, 서브 에이전트는 "그 매뉴얼을 숙지한 담당자"입니다. 도구를 직접 쓰는 것과 담당자에게 맡기는 것의 차이입니다.

설치 — VS Code + Claude Code 확장
~10분
▶
터미널(검은 화면)이 익숙하지 않다면 VS Code 확장 방식이 훨씬 편합니다.
VS Code는 파일 탐색기처럼 생긴 코드 편집기인데, Claude Code를 사이드바에서 채팅하듯 쓸 수 있습니다. Anthropic 공식 권장 방식입니다.
① VS Code 설치 (이미 있으면 건너뜀)
🔗
code.visualstudio.com 접속 → Download → 설치. Windows/Mac 모두 동일.
② Claude Code 확장 설치
🔍
VS Code 실행 → 왼쪽 사이드바 Extensions 아이콘(네모 4개) 클릭 → 검색창에 "Claude Code" 입력 → Anthropic이 만든 공식 확장 → Install
③ Claude 로그인
🔑
설치 후 왼쪽 사이드바에 ✦ (스파크) 아이콘 클릭 → Claude 계정으로 로그인 → 완료.
계정 없으면 claude.ai에서 무료 가입 후 Pro 플랜($20/월) 필요.
💡
확장이 안 보이면 VS Code를 재시작하거나, Cmd+Shift+P(Mac) / Ctrl+Shift+P(Windows) → "Developer: Reload Window" 실행.
④ 세팅 상태 확인 — 첫 번째 입력
VS Code 사이드바 Claude 채팅창에 붙여넣기

결과 활용
결과물, 어떻게 저장하고 쓰나요?

/research, /prd 같은 명령어를 실행하면 결과가 대화창에 텍스트로 출력됩니다.
자동으로 파일이 저장되지는 않습니다. 아래 방법 중 하나를 선택하세요.

📁
결과 나온 후 파일 저장 요청 — 가장 깔끔
결과를 확인하고 마음에 들면 바로 저장을 요청합니다.
결과 나온 후 바로 입력
방금 결과를 바탕화면에 파일로 저장해줘. 파일명: research-결과.md
📋
처음부터 저장 포함해서 요청
명령어 실행 시 저장 위치를 함께 지정합니다.
저장까지 한 번에
/research 국내 (업종) 경쟁사 분석 완료되면 결과를 ~/Desktop/research-(업종).md 파일로 저장해줘.
📝
대화창에서 복사 후 붙여넣기

"~/Desktop/claude-outputs/ 폴더에 저장해줘"라고 하면 됩니다.

추천 설정
플랜 모드를 기본값으로 쓰는 걸 추천합니다

플랜 모드란 Claude Code가 작업을 실행하기 전에 먼저 계획을 보여주고, 승인하면 실행하는 방식입니다.
처음 Claude Code를 쓰는 분이라면 기본값으로 켜두는 걸 권장합니다.

플랜 모드의 장점
처음 쓰는 사람일수록 더 유용합니다
👁️
Claude가 뭘 할지 먼저 보입니다
실행 전에 "이렇게 하겠습니다"를 보여줍니다. 예상과 다르면 수정하고, 맞으면 승인. 실수를 미리 막을 수 있습니다.
🧠
Claude Code를 이해하는 속도가 빨라집니다
"아, Claude가 이렇게 생각하는구나"를 반복해서 보다 보면 어떻게 지시해야 더 좋은 결과가 나오는지 자연스럽게 익혀집니다.
🔒
중요한 파일을 실수로 수정하는 걸 막습니다
파일을 여러 개 만들거나 수정하는 작업 전에 계획을 확인하고 진행하기 때문에 의도하지 않은 변경을 방지합니다.
✏️
계획 단계에서 방향을 바꿀 수 있습니다
실행 후 수정보다 계획 단계에서 수정이 훨씬 빠릅니다. "3번 방법 말고 2번 방법으로 해줘"처럼 자연스럽게 조정 가능합니다.
켜는 방법 — 3가지
⌨️
방법 1 — 키보드 단축키 (가장 빠름)
Claude Code 입력창에서 Shift + Tab 을 두 번 누르면 플랜 모드로 전환됩니다.
화면 하단에 ⏸ plan mode on 표시가 뜨면 켜진 것입니다.
💬
방법 2 — 명령어 입력
입력창에 /plan 입력 후 Enter. 작업 요청과 함께 바로 쓸 수도 있습니다.
⚙️
방법 3 — 기본값으로 설정 (매번 켜기 귀찮을 때)
Claude Code 입력창에 아래 명령어를 입력하면 설정 파일이 자동으로 바뀝니다.
/config
/config 실행 → "Default permission mode" → "plan" 선택
또는 CLAUDE.md에 "모든 작업 전 계획 먼저 보여주고 승인 후 실행해줘" 추가
⚠️ Windows 환경에서는 /config 설정이 적용 안 될 수 있습니다. 그 경우 Shift+Tab 두 번으로 직접 켜세요.
익숙해지면 끄면 됩니다. 플랜 모드는 Claude Code를 처음 배우는 단계에서 가장 유용하고, 능숙해질수록 빠른 실행이 더 편해집니다.
