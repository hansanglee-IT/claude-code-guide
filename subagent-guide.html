개념 정리
Skills와 서브 에이전트, 무엇이 다른가요?

1단계에서 만든 Skills는 훌륭한 도구입니다. 그런데 한 단계 더 나아가고 싶을 때가 있습니다.
"도구를 쓰는 것"과 "담당자에게 맡기는 것"의 차이입니다.

1단계 — Skills
도구
내가 직접 "/research"를 입력하면 정해진 형식대로 작업합니다. 매뉴얼이 있는 도구를 내가 꺼내 쓰는 방식입니다.
예시 /research → 리서치 보고서 형식으로 작성
/prd → PRD 양식으로 작성
2단계 — 서브 에이전트
역할을 가진 팀원
리서치팀, 기획팀처럼 역할과 전문성을 가진 Claude입니다. 독립된 공간에서 작업하고 결과만 보고합니다.
예시 @리서치팀 → 리서치 전문가로서 분석
@기획팀 → 기획자 관점에서 PRD 작성
💡
핵심 차이: 독립된 컨텍스트 서브 에이전트는 메인 대화창과 별도의 작업 공간에서 실행됩니다. 무거운 리서치나 복잡한 분석을 맡겨도 메인 대화가 복잡해지지 않습니다. 결과만 깔끔하게 받아볼 수 있습니다.
설치 가이드
설치 방법 2가지

빠른 시작: 아래 한꺼번에 설치 프롬프트로 8개 팀을 한 번에 만드세요.
정교한 구성: 설치 후 아래 「AI 팀 8개」 섹션의 각 팀 개별 프롬프트로 덮어쓰면 더 세밀하게 설정됩니다.

~10분
▶

이 방법은 빠른 시작용입니다. Claude가 각 팀의 역할을 자체적으로 판단해서 만들기 때문에 퀄리티가 다소 다를 수 있습니다.
각 팀을 더 정교하게 구성하시려면 아래 「스타트업 대표를 위한 AI 팀 8개」 섹션의 개별 프롬프트를 활용해주세요.
(업종)만 바꿔서 붙여넣기
~/.claude/agents/ 폴더에 아래 8개 서브 에이전트를 만들어줘. 내 업종: (업종) [1] research-team — 시장 리서치, 경쟁사 분석 [2] pm-team — PRD 작성, 기능 스펙 [3] dev-team — 기능 구현, 코드 리뷰, 보안 점검 [4] legal-team — 계약서 사전 스크리닝 [5] ir-team — 투자자 문서, IR 자료 [6] comm-team — 비즈니스 이메일, 파트너 소통 [7] finance-team — 지출 정리, 세무 자료 준비 [8] hr-team — 채용, 면접, 온보딩, 평가 각자 역할, 아웃풋 형식, 규칙 포함해서 전문적으로 만들어줘. 완료 후 전체 목록 보여줘.

2단계 — 실제 작동 테스트
에이전트가 실제로 위임되는지 확인하려면 아래처럼 요청합니다. Claude가 리서치 에이전트를 호출해서 처리하는 과정이 보이면 성공입니다.
리서치팀 작동 테스트 — 업종 넣어서
리서치 에이전트를 사용해서 국내 (업종) 시장 주요 경쟁사 3개를 비교해줘
💡
에이전트가 작동하지 않는다면 — Claude Code에 "설치된 에이전트 목록 보여줘"라고 입력하세요. Claude가 ~/.claude/agents/ 폴더를 확인하고 목록을 알려줍니다. 없다면 위 설치 프롬프트를 다시 실행해보세요.

상시

팀 구성
스타트업 대표를 위한 AI 팀 8개

각 팀을 클릭하면 역할 설명과 개별 설치 프롬프트를 확인할 수 있습니다.
한꺼번에 설치 후 특정 팀을 더 정교하게 만들고 싶다면 아래 개별 프롬프트를 붙여넣으세요.
이미 설치된 팀이 있어도 괜찮습니다. 같은 파일명으로 덮어쓰기 되므로 기존 파일은 업그레이드됩니다.

에이전트를 정확하게 부르는 방법

Claude는 요청 내용의 키워드를 각 에이전트 description과 매칭해서 자동으로 위임합니다.
키워드가 겹치거나 애매하면 엉뚱한 에이전트가 호출될 수 있습니다. 역할을 명시하면 훨씬 정확합니다.

⚠️ 애매한 요청 — 오판 가능
이번 달 매출 분석해줘
재무 현황 정리해줘
비용 관련해서 알아봐줘
✅ 역할 명시 — 정확하게 위임
회계 담당으로 이번 달 지출 분류해줘
IR 자료용으로 재무 현황 정리해줘
리서치 에이전트로 경쟁사 비용 조사해줘
VS Code 환경 참고: 에이전트 목록을 UI에서 직접 확인하거나 단축키로 지목하는 기능은 현재 VS Code 확장에서 지원되지 않습니다. 역할을 명시해서 요청하는 습관이 가장 확실한 방법입니다. 엉뚱하게 작동했다면 "방금 작업은 회계팀 역할로 다시 해줘"라고 수정 요청하면 됩니다.

리서치팀
시장 분석 · 경쟁사 조사 · 트렌드 리포트
▶
시장 조사, 경쟁사 분석, 업계 트렌드 파악을 전담합니다. 단순히 정보를 나열하는 것이 아니라 "우리 회사에 어떤 의미인가"까지 해석해서 보고합니다.
이렇게 말하면 됩니다 — Claude가 자동으로 위임 국내 부동산 SaaS 경쟁사 5개 비교해줘. 각사 주요 기능, 가격, 타깃 고객 포함해서.
요즘 우리 업종 투자 트렌드 어떻게 돼? 최근 6개월 기준으로 정리해줘.
우리 경쟁사 A사가 최근에 뭔가 바뀐 게 있는지 조사해줘.

⚠️
알아둘 것 3가지

1. 내 컴퓨터 파일은 못 읽습니다 — 원격 서버에서 실행되기 때문에 로컬 파일 접근 불가. 프롬프트에 필요한 정보를 모두 포함해야 합니다.
2. 이전 대화 맥락이 없습니다 — 매번 새 세션에서 실행됩니다. "아까 그거"는 통하지 않으니, 프롬프트가 자체 완결적이어야 합니다.
3. 최소 간격은 1시간입니다 — 5분마다, 30분마다 같은 짧은 간격은 지원하지 않습니다.

참고 — 고급 설정
tools 필드 — 에이전트가 할 수 있는 것 제한하기

지금 설치한 에이전트들은 기본적으로 Claude Code가 할 수 있는 모든 것을 할 수 있습니다. 처음에는 이게 더 편합니다.
나중에 "이 에이전트는 파일을 읽기만 하고 수정은 하지 말았으면 좋겠다"는 경우가 생기면 tools 필드로 제한할 수 있습니다.

주요 tools 종류 — 비개발자 기준
Read 파일 읽기만 가능. 계약서나 문서를 분석할 때 수정 없이 읽기만 하게 하고 싶을 때.
Write 파일 생성·수정 가능. 문서를 만들거나 저장할 때 필요.
WebSearch 웹 검색 가능. 리서치팀처럼 인터넷에서 정보를 찾아야 할 때.
Bash 터미널 명령어 실행 가능. 비개발자는 보통 안 써도 됨.
적용 예시
# 법무팀 — 읽기만, 파일 수정 불가
tools: Read, WebSearch

# 리서치팀 — 읽기 + 웹검색
tools: Read, WebSearch

# 기획팀 — 읽기 + 파일 작성
tools: Read, Write

# tools 없음 = 제한 없음 (기본값, 지금 설치된 상태)
적용 방법 — 그냥 말로 하면 됩니다
예시 — 법무팀에 tools 제한 추가
법무팀 에이전트 설정에 tools: Read, WebSearch 추가해줘. 파일을 읽고 웹검색만 하고 수정은 못하게 하는 거야.
언제 필요한가 혼자 쓸 때는 제한 없이 쓰는 게 편합니다. "에이전트가 실수로 중요한 파일을 수정할까봐 걱정된다"거나 팀원과 공유할 때 추가하세요. 처음부터 설정할 필요는 없습니다.
