개념 정리
Skills와 서브 에이전트, 무엇이 다른가요?

1단계에서 만든 Skills는 훌륭한 도구입니다. 그런데 한 단계 더 나아가고 싶을 때가 있습니다.
"도구를 쓰는 것"과 "담당자에게 맡기는 것"의 차이입니다.

1단계 — Skills
도구
내가 직접 "/research"를 입력하면 정해진 형식대로 작업합니다. 매뉴얼이 있는 도구를 내가 꺼내 쓰는 방식입니다.
예시 /research → 리서치 보고서 형식으로 작성
/prd → PRD 양식으로 작성
2단계 — 서브 에이전트
역할을 가진 팀원
리서치팀, 기획팀처럼 역할과 전문성을 가진 Claude입니다. 독립된 공간에서 작업하고 결과만 보고합니다.
예시 @리서치팀 → 리서치 전문가로서 분석
@기획팀 → 기획자 관점에서 PRD 작성
💡
핵심 차이: 독립된 컨텍스트 서브 에이전트는 메인 대화창과 별도의 작업 공간에서 실행됩니다. 무거운 리서치나 복잡한 분석을 맡겨도 메인 대화가 복잡해지지 않습니다. 결과만 깔끔하게 받아볼 수 있습니다.

tools 필드 — 에이전트 권한 제한하기

지금 설치한 에이전트들은 기본적으로 Claude Code가 가진 모든 도구를 쓸 수 있습니다. 처음에는 이게 더 편합니다.
나중에 여러 팀원이 함께 쓰거나 더 엄격한 제어가 필요할 때 tools 필드로 권한을 제한할 수 있습니다.

tools 필드 예시
# 읽기만 가능 (파일 수정 불가)
tools: Read, Grep

# 읽기 + 웹 검색
tools: Read, WebSearch

# 읽기 + 쓰기 (파일 생성/수정 가능)
tools: Read, Write

# 제한 없음 (기본값)
# tools 필드를 아예 쓰지 않으면 모든 도구 사용 가능
언제 tools 제한이 필요한가 혼자 쓸 때는 제한 없이 쓰는 게 편합니다. 팀원과 함께 공유하거나, 에이전트가 실수로 파일을 수정하는 게 걱정될 때 제한을 추가하세요. 법무팀처럼 "읽고 분석만" 해야 하는 에이전트에 tools: Read를 주는 식으로 활용하면 됩니다.
